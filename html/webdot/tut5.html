Similarly to the support for client-side maps, WebDot can filter an html page and replace all occurrences of <webdot...> tags with a client-side SVG image. For example, tut5.dot contains: 
    graph G {
        node [URL="/webdot/\N.html"]
        a -- b -- c; b -- d;
    }
and the source of this page contains (for the graph on the right): 
    <webdot src=/webdot/tut5.dot engine=neato type=svg>

after serving this page through /cgi-bin/webdot, this <webdot...> tag is replaced by: 

    <!-- Generated by dot version 1.7.11 (Tue Oct 23 10:56:32 EDT 2001)
         For: (root) root
         Title: G
         Pages: 1
    -->
    <svg width="148px" height="202px" xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'>
    <g id="G" class="graph">
    <g id="a" class="node">
    <a xlink:href="/webdot/a.html">
    <ellipse cx="74" cy="29" rx="27" ry="18" style="fill:none;stroke:black"/>
    <text x="71" y="33" style="font-family:Times;font-size:14.00">a</text>
    </a></g>
    <g id="b" class="node">
    <a xlink:href="/webdot/b.html">
    <ellipse cx="74" cy="101" rx="27" ry="18" style="fill:none;stroke:black"/>
    <text x="70" y="105" style="font-family:Times;font-size:14.00">b</text>
    </a></g>
    <g id="a->b" class="edge">
    <g style="fill:none;stroke:black"><path d="M74 47C74 58 74 72 74 83"/></g>
    </g>
    <g id="c" class="node">
    <a xlink:href="/webdot/c.html">
    <ellipse cx="38" cy="173" rx="27" ry="18" style="fill:none;stroke:black"/>
    <text x="35" y="177" style="font-family:Times;font-size:14.00">c</text>
    </a></g>
    <g id="b->c" class="edge">
    <g style="fill:none;stroke:black"><path d="M65 118C60 130 52 144 47 156"/></g>
    </g>
    <g id="d" class="node">
    <a xlink:href="/webdot/d.html">
    <ellipse cx="110" cy="173" rx="27" ry="18" style="fill:none;stroke:black"/>
    <text x="106" y="177" style="font-family:Times;font-size:14.00">d</text>
    </a></g>
    <g id="b->d" class="edge">
    <g style="fill:none;stroke:black"><path d="M83 118C88 130 96 144 101 156"/></g>
    </g>
    </g></svg>

The advantages of this mode of WebDot are the much simpler authoring and the responsiveness of client-side mapping.

The disadvantage is that the page has to be served through the /cgi-bin/webdot program, and that care must be taken that relative URLs in the html don't also get served through the program. It is important to use "rooted" hrefs in documents served this way.
